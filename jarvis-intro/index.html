scene 01 / why jarvis
AI vibe coding tools

AI vibe coding 很快
但常常停在第一版

很多工具很會生畫面、寫 code、做 demo。
但新手真正卡住的，通常不是第一版，而是做完第一版以後，接下來到底怎麼走。

Jarvis 的重點不是只給你一段答案，而是把整件事接住
第一版很快 一下子就有畫面、有 code、有一個看起來像樣的結果。
但下一步常常沒有人接
改到一半會亂 需求一變、問題一多，容易回到重新問一次、重新做一次。
新人最容易卡住 不是不會問，而是不知道怎麼把一個模糊想法推成真正完成。
Jarvis 想解的不是「生成一段東西」，而是「幫你把工作一路推進到可交付」。
scene 02 / system
Jarvis runtime

Jarvis 不只是聊天工具
它是整套 AI 工作系統

它不是只回答你一句，而是把不同角色、能力、流程和記憶串起來，讓事情能往前走。

A Agents 不同專長的 AI 角色，負責拆問題、執行工作、接力合作。
S Skills 可重用的能力模組，像設計、研究、寫作、工具操作。
W Workflows 固定流程直接跑，不用每次都從頭想 prompt。
M Memory 把任務狀態記下來，讓 AI 不會每次都像第一次見面。

from human ask to real work

你不需要先想好專業 prompt。先把目標講清楚，Jarvis 會幫你整理方向、選工具、安排下一步。

你只要先講需求

1
理解真正目標 先分辨你是要展示、銷售、教學，還是要直接拿去交付。
2
挑對 agent 和 skill 把適合的能力接進來，不用你自己猜哪個工具比較對。

scene 04 / memory
continuity matters

Jarvis 會記得
工作進度，不只記得聊天內容

它不是只記一句對話，而是記住任務在哪裡、做到哪裡、交接過什麼，所以下次可以直接續做。

記住的是整個任務 從任務狀態、交接摘要，到可重用 know-how，都會留下來，讓後續工作更快更穩。
不是每次重來，是每次接著做
session-state 這次任務目前做到哪裡，哪些事情已經完成。
handoff 如果中途換人或換階段，下一個 agent 能快速接手。
context-window 全域重點摘要，方便快速理解現在的大局。
auto-skill 把做過的好方法留下來，下次同類任務直接複用。
